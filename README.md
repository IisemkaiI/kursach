## Описание
Реляционная база данных для автоматизации складского учёта одного склада в СУБД **MySQL 8.0+**.
Реализует приёмку, размещение, хранение, перемещение и отгрузку товаров с журналом операций
и автоматическим пересчётом остатков.

## Назначение
- ведение справочников товаров, категорий, поставщиков и зон хранения;
- учёт остатков товаров по зонам;
- регистрация складских операций (приход / расход / перемещение);
- разграничение прав доступа (admin / storekeeper).

## Структура БД (7 таблиц, 3НФ/BCNF)
`category`, `supplier`, `product`, `zone`, `stock`, `operation`, `app_user`.

## Объекты БД
- Триггер `trg_operation_after_insert` — автообновление остатков;
- Функция `fn_stock_value(product_id)` — стоимость остатков товара;
- Представление `v_stock_report` — отчёт об остатках;
- Процедура `sp_receive_product(...)` — приёмка товара;
- Процедура `sp_move_product(...)` — перемещение между зонами (транзакция);
- Роли `admin_role`, `storekeeper_role`.

## Типовые запросы (примеры)
```sql
-- Отчёт об остатках
SELECT * FROM v_stock_report;

-- Стоимость остатков товара #1
SELECT fn_stock_value(1);

-- Уровень запасов (CASE)
SELECT p.name, s.quantity,
  CASE WHEN s.quantity<20 THEN 'Критический'
       WHEN s.quantity<100 THEN 'Низкий'
       ELSE 'Достаточный' END AS stock_level
FROM stock s JOIN product p ON p.product_id=s.product_id;

-- Приёмка и перемещение
CALL sp_receive_product(2,2,2,50,@r); SELECT @r;
CALL sp_move_product(2,2,3,2,30,@r);  SELECT @r;
```

## Технические требования для запуска
- MySQL Server 8.0+;
- кодировка `utf8mb4`, движок `InnoDB`;
- клиент MySQL Workbench / CLI.
